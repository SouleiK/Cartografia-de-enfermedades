1 Modelizacin con INLA

1.1 Introduccin

En las conclusiones de la primera parte del trabajo comentbamos como al incluir en el modelo jerrquico la dependencia espacial, el efecto aleatorio por municipio y el suavizado con la funcin CAR obtenamos un modelo verosmil. Pero la propia virtud de los mtodos de aproximacin por MCMC, que es su versatilidad, nos hace encontrarnos con el problema principal de este tipo de modelos, que es el tiempo de computacin.

En concreto nuestro modelo con WinBUGS ha necesitado entre 150 y 250 segundos para completarse. Contando con que este modelo es sencillo como ejemplo de clase, es fcil imaginarse lo rpido que escala el consumo de tiempo de procesado de los mtodos MCMC (WinBUGS, JAGS, OpenBUGS).

Para aquellos modelos que pueden incluirse en la categora de Latent Gaussian Models existe una alternativa rpida, la aproximacin por INLA ( Integrated Nested Laplace Approximation ).

1.2 Modelando

Al igual que con los modelos para WinBUGS, el primer paso es establecer la dependencia espacial mediante la creacion de una red de vecindad, una matriz de ID para tener en cuenta el efecto aleatorio del municipio, establecemos la frmula que usaremos en el modelo y lo lanzamos:

veci <- poly2nb(aragon)
nb2INLA("datos/ar.graph", veci)

H <- inla.read.graph(filename = "datos/ar.graph")
image(inla.graph2matrix(H), xlab = "", ylab = "")
plot(H)

S <- U <- seq(1, 729)
Aragon.df <- cbind(Aragon.df, S, U)

formula <- O ~ 1 + f(S,
  model = "besag",
  graph = H,
  scale.model = TRUE,
  hyper =
    list(prec = list(prior = "loggamma", param = c(1, 0.001)))
) +
  f(U,
    model = "iid",
    hyper =
      list(prec = list(prior = "loggamma", param = c(1, 0.001)))
  )

tic("Modelo INLA")
model <- inla(formula,
  family = "poisson",
  data = Aragon.df,
  E = E,
  control.compute = list(dic = TRUE, waic = TRUE, cpo = TRUE),
  control.predictor = list(compute = TRUE, cdf = c(log(1)))
)

aragon$RMEmean <- round(model$summary.fitted.values$mean, 4)
aragon$RMEsd <- round(model$summary.fitted.values$sd, 5)
